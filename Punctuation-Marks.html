Punctuation Marks

Parentheses () : grouping expressions, function calls, passing arguments.

Curly braces {} : defining blocks of code as function and loop bodies.

Square Brackets [] : indexing and accessing elements in arrays or lists.

Angle Brackets <>: generic type parameters and template arguments.

Semicolon ; : terminate statements, indicating the end of an instruction.

Comma , : separate items in a list, arguments in function calls, or elements in an array.

Quotation Marks " " : represent strings, which are sequences of characters.

Single Quotation Marks ' ' represent characters or single-character strings.

Backtick ' : string interpolation or defining template literals.

Period . : a delimiter to access properties or methods of objects or to reference nested elements.

Colon : : defining a dictionary or object literal, specifying type annotations, or indicating a label in some programming languages.

Exclamation Mark ! : for logical negation or to denote not equal to in some programming languages.

Question Mark ? : conditional expressions, optinal values, or to indicate the presence of a method or property.

Slash /a division operator, to represent paths or regular expressions, or to indicate line comments in some programming languages.

Asterisk * : multiplication, exponentiation, or as a wildcard character in some contexts.

Ampersand & : for logical conjunction (AND) or bitwise operations in some pl.

Pipe | : for logical disjunction (OR) bitwise operations in some pl.

Equals Sign = : for assignment, variable initialization, or comparision (equality) in some pl.

Plus Sign + : for addition or concatenation of strings or arrays.

Minus Sign -for subtraction or negation.
